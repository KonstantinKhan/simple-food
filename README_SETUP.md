# Быстрая настройка проекта

## Что было сделано

Созданы файлы конфигурации для VS Code/Cursor:
- `.vscode/extensions.json` - список рекомендуемых расширений
- `.vscode/settings.json` - настройки проекта

## Что нужно сделать СЕЙЧАС

### 1. Установите расширения

VS Code/Cursor должен автоматически предложить установить расширения. Если нет:

1. Нажмите `Ctrl+Shift+X` (открыть панель расширений)
2. Введите "Kotlin Language" и установите
3. Введите "Gradle for Java" и установите  
4. Введите "Extension Pack for Java" и установите

**Или используйте команду:**
- `Ctrl+Shift+P` → "Extensions: Show Recommended Extensions" → "Install All"

### 2. Обновите Gradle проект в IDE

После установки расширений:

1. Нажмите `Ctrl+Shift+P`
2. Введите "Gradle: Refresh Gradle Project"
3. Дождитесь завершения синхронизации (может занять 1-2 минуты)

### 3. Очистите кэш IDE (если ошибки остаются)

Если ошибки в IDE все еще появляются:

1. Нажмите `Ctrl+Shift+P`
2. Введите "Java: Clean Java Language Server Workspace"
3. Подтвердите перезагрузку окна
4. После перезагрузки: `Ctrl+Shift+P` → "Gradle: Refresh Gradle Project"

## Почему появляются ошибки в IDE?

Это нормально! IDE может показывать ошибки, которые не влияют на сборку проекта. Причины:

1. **IDE еще не синхронизировался с Gradle** - нужно обновить проект (шаг 2 выше)
2. **Старый кэш IDE** - нужно очистить кэш (шаг 3 выше)
3. **IDE не понимает некоторые Gradle синтаксисы** - это не критично, если проект собирается

## Проверка

После выполнения шагов:

1. Откройте любой `.kt` файл
2. Убедитесь, что есть подсветка синтаксиса
3. Попробуйте автодополнение (`Ctrl+Space`)
4. Запустите сборку: `./gradlew build` (в терминале)

Если проект собирается в терминале - все работает правильно, даже если IDE показывает ошибки.

## Полезные команды

- `Ctrl+Shift+P` → "Gradle: Refresh Gradle Project" - обновить проект
- `Ctrl+Shift+P` → "Java: Clean Java Language Server Workspace" - очистить кэш
- `Ctrl+Shift+P` → "Gradle: Run Gradle Task" - запустить задачу Gradle
